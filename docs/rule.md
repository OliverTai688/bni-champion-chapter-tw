# BNI 長冠軍分會 — 座位排列規則手冊

> 此文件為 Claude 排座位的唯一規則來源。每次收到排座位指令時，應先查閱本文件確認規則，再建立新的 `layout-MMDD.ts`。

---

## 一、實體場地對應關係

會場有**兩排長桌**，座位表的 4 欄對應如下：

```
┌─────────────────────────┬─────────────────────────┐
│     第一排長桌（Table 1）   │     第二排長桌（Table 2）   │
│   欄 0（左）│ 欄 1（右）   │   欄 2（左）│ 欄 3（右）   │
└─────────────────────────┴─────────────────────────┘
```

- 同一張長桌的兩欄成員（欄0+欄1 或 欄2+欄3）坐在桌子兩側**面對面**。
- 「坐在一起」= 同欄 pair（同一張桌子）或連續相鄰的列。
- 生成 layout 時，希望**同產業服務鍊組（A/B/C/D）的成員分配在同一張長桌**，跨桌越少越好。

## 三、音控 & 值日生

- 每週由指令指定人選。
- **固定位置（有來賓週 / 無來賓週皆同）**：
  - 值日 → **列0・欄3**（第二排長桌，欄3，最前列）
  - 音控 → **列1・欄3**（第二排長桌，欄3，第二列，即值日生正後方）
  - 實測依據：0625（有來賓）確認此配置為正確版本。
- 無論位置如何，`isSound: true` / `isDuty: true` 旗標必須標記。

## 四、來賓 & 執事（有來賓週）

### 4-1 排列規則
1. 來賓**盡量靠前坐**（列號最小）。
2. 執事坐在來賓的**正後方**（同欄，下一列）。
3. **避開值日與音控座位**：欄 3 的前兩列（列0・欄3 及 列1・欄3）已固定保留給值日與音控，來賓與執事不可安排在此。
4. 賓號排列慣例（由左至右、由上而下，避開欄3保留位）：
   ```
   列 0：賓1 │ 賓2 │ 賓3 │ 值日
   列 1：執1 │ 執2 │ 執3 │ 音控
   列 2：賓4 │ 一般 │ 一般 │ 一般
   列 3：執4 │ 一般 │ 一般 │ 一般
   ```
5. 若來賓數量較多，依序往下列填入。
6. `isGuest: true`，`isHost: true` + `hostFor: '賓X'` 旗標必須標記。

### 4-2 來賓人數對應列數
| 來賓數 | 佔用列數 | 說明 |
|--------|----------|------|
| 0 | 0 | 無來賓，直接從一般成員開始排 |
| 1-4 | 2 | 賓1-4 在列0，執事在列1 |
| 5-8 | 4 | 再加列2（賓5-8）與列3（執事） |
| 9 | 6 | 賓9 單獨佔一格，執9 在下一列 |

## 五、一般成員（A/B/C/D 產業服務鍊分組）

### 5-1 分桌原則
- 同組成員優先分配在**同一張長桌**（同欄 pair）。
- A/B/C/D 四組，兩組分 Table 1，兩組分 Table 2。

## 六、代理人（Proxies）

- 固定排在**最後一列**。
- 格式：`null │ 代理1 │ 代理2 │ null`（兩側留空，置中顯示）。
- 若只有 1 位：`null │ null │ 代理1 │ null` 或 `null │ 代理1 │ null │ null`。
- `role: '代理'` 旗標必須標記。

---

## 七、英雄榜

- `heroes: []` → 英雄榜區塊不顯示。
- `heroes: ['名字1', '名字2', ...]` → 顯示英雄榜，依英1→英N順序。
- 名字可用**暱稱**（如「孟婷」而非「邱孟婷」）。
- 無英雄榜週：`heroes: []`，`initialHeroes={ROSTER_MMDD.heroes}` 傳入元件。

## 八、Layout 檔案規範

### 8-1 命名與位置
- 路徑：`src/lib/layout-MMDD.ts`（MMDD = 月日，例 `layout-0506.ts`）
- 每週建立**新檔案**，絕不覆蓋舊檔。

### 8-2 必要 Export
```typescript
export const ROSTER_MMDD: Roster = { ... }   // 含 heroes, guests, proxies 等
export const LAYOUT_MMDD: SeatingLayout = { topRoles, mainGrid, sidebar }
```

### 8-3 更新 page.tsx
每週更新 `src/app/seats/page.tsx`：
```typescript
import { LAYOUT_MMDD, ROSTER_MMDD } from '@/lib/layout-MMDD';
// ...
<SeatingArranger initialLayout={LAYOUT_MMDD} initialHeroes={ROSTER_MMDD.heroes} />
```
標題也要更新為當週日期（格式：`115/MM/DD 座位表`）。

## 九、行政小組（第1-6組）vs 產業服務鍊（A/B/C/D）

這是**兩套不同的分組系統**：

| 系統 | 標籤 | 用途 | 來源 |
|------|------|------|------|
| 產業服務鍊 | A1~D4 | 座位卡標籤、分桌依據 | `industry-chains.ts` |
| 行政小組 | 第1~6組 | 競賽/行政分組 | `administrative_grouping_report.md` |

座位排列以**產業服務鍊（A/B/C/D）**為主要分組依據。

## 十、快速指令對照表

| 用戶說 | Claude 應做 |
|--------|------------|
| 「這週值日生：XXX」 | `duty: 'XXX'`，`isDuty: true` 的格子填入 XXX |
| 「音控：XXX」 | `sound: 'XXX'`，`isSound: true` 的格子填入 XXX |
| 「代理人：A、B」 | `proxies: ['A','B']`，最後列 null\|A\|B\|null |
| 「沒有英雄榜」 | `heroes: []` |
| 「英雄榜：A、B、C」 | `heroes: ['A','B','C']` |
| 「沒有來賓」 | `guests: []`，移除所有 `isGuest/isHost` 格子 |
| 「N 位來賓，賓X 執事是 Y」 | 建立 GuestPair，依四-一規則排列 |
| 「A/B/C/D 同組坐在一起」 | 同組排同 column pair（同一張長桌） |
| 「新增 XXX，是 Y 組標籤」 | 加入 mainGrid + 加入 `industry-chains.ts` 對應鍊 |
