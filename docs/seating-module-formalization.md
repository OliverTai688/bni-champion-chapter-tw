# 排座模組正式化報告

## 正式模組目標

排座模組要支援 BNI 長冠軍分會管理成員完成每週例會座位安排，並具備後續接入會員、商務關係、例會品質與活動管理模組的資料邊界。

## 已實作能力

### 工作台

- 顯示當週週次、分會與例會情境。
- 支援拖拉座位。
- 顯示座位人數、來賓、代理、英雄榜與規則分數。
- 顯示未保存狀態與最後保存時間。
- 直接顯示規則基準，讓管理者不必離開工作台即可查看排座規則。
- 顯示分會成員列表與本週固定出席成員對照。

### 規則編輯

- 編輯主持團。
- 編輯音控與值日生。
- 新增、編輯、移除來賓與對應執事。
- 新增、編輯、移除固定出席成員；此名單不因拖拉座位改變。
- 新增、編輯、移除代理人。
- 新增、編輯、移除英雄榜。
- 編輯產業服務鍊成員。

### 座位套用

- 既有座位會依角色更新姓名。
- 新增來賓會尋找空位，並建立來賓與執事上下對應座位。
- 新增一般成員與代理人會補入空位；空位不足時自動擴充完整列。

### 規則檢查

- 主持團應為 5 人。
- 主持團姓名不可空白。
- 音控與值日生應各 1 人。
- 音控目前基準位置為列 1・欄 3；值日生目前基準位置為列 0・欄 3。
- 值日生正後方不可為來賓。
- 來賓與執事應上下對應。
- 來賓位置偏後會提示。
- 代理人未集中最後列會提示。
- 產業服務鍊 A/B/C/D 跨桌分布會提示。

### 匯出與列印

- CSV 匯出包含週次、分會、英雄榜、主持團、座位座標、角色與產業服務鍊。
- PDF 透過專屬列印路由產生，讀取同一份 workspace state。

### 儲存

- 本機草稿使用 `localStorage`。
- repository 介面已抽出，未來可替換為 Supabase/Prisma 實作。

## 尚未進入正式模組的限制

這些限制不阻擋第一節點，但應列入下一階段：

- 尚未建立多週次管理 UI。
- 尚未接會員主檔，因此姓名仍是文字輸入。
- 尚未有登入與角色權限。
- 尚未將規則檢查寫成測試案例。
- 尚未支援由 CSV 直接產生新週座位。

## 第一節點判定

可視為正式模組的最低標準：

- 管理者可以完成本週排座。
- 管理者可以維護本週名單。
- 系統能提示規則風險。
- 系統能保存草稿。
- 系統能輸出 CSV 與 PDF。
- 程式通過 lint、TypeScript 與 production build。

當以上條件全部成立，即達成第一節點：「排座位功能可以上架成為正式模組」。
